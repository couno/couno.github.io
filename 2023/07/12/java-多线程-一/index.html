Java多线程-教程(一)-开始线程世界

1.简述

Java 给多线程编程提供了内置的支持。 一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。

这里是静态代理，在之前的实现Runnable接口的时候，出现了

new Thread(对象).start()；// 这其实就是静态代理

Thread作为代理者，被代理的对象只需要关注输入与输出就可以

下面是一个静态代理的实例：（这里学习自狂神编程多线程案例）

目标简述：一个人（person）要结婚（Merry），他希望他将所有的事务交给婚庆公司（WeddingCompany）去处理，他只要到时候出席自己的婚礼就可以了。

在这里，婚庆公司就是一个代理者，Merry是目标，实际者是person

这段代码也是很好理解

  • 首先创建一个Merry接口
  • 创建一个person类，并且实现Merry接口（那么他将拥有happyMerry的方法），重写happyMerry方法
  • 创建一个WeddingCompany类，也实现Merry接口，重写happyMerry方法。但是在个类中，定义了一个Merry类型的变量target，也就是结婚的主人公person。
    在重写happyMerry的时候，调用this.target.happyMerry()方法，也就是结婚动作
    • 但这里婚庆公司不能什么都不做吧，所以在结婚前有before动作，结婚后又after动作
    • 这里就是一个很重要的概念，我的person只需要关心我的主要事件（Merry）不需要去操办什么场地呀，音乐呀，人员和上菜的顺序……..。这些都是代理做的
    • 我们只需要关心的是输入与输出

5.左接口，右实现

在这里我对左接口，右实现类更加深入的感悟

例如：

在之前，我对这样的写法，也就是多态的感觉。但是对它这样用的好处感官不是很大。

首先，这样写的目的。

  • 第一：多态性,这里可以想象一下，如果你左边是实现类，右边是实现对象，那么如果我哪一天想要将ArrayList换成LinkedList，那么我就需要将左边的实现类，也换成LinkedList。
    但是我如果用多态的思维来写呢，左边是List，右边是ArrayList，我哪一天想要改的时候，我直接可以修改ArrayList换成LinkedList

    这里有个很重要的前提，那就是必须都要实现了相同的接口

  • 第二：松耦合,上述已经可以解释一部分，类之间的联系降低，更加利于维护和扩展。通过使用接口作为类型，将代码与特定的实现类解耦。这意味着你可以更轻松地更改或替换底层的实现类，而无需修改使用该接口的其他代码。这种松耦合的设计有助于提高代码的可维护性和可扩展性。

  • 第三：抽象性：接口提供了对实现类的抽象表示。通过编写针对接口而不是具体实现的代码，你可以关注于对象应该提供的功能，而不用过多关注实现的细节。这样可以使代码更加清晰和易于理解
